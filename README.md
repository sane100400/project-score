# PROJECT:SCORE

> IT 프로젝트 주제를 시작하기 전에, 이 아이디어가 정말 시작해도 될 만큼 준비됐는지 스스로 진단하는 **정적 체크리스트**.

체스판 톤의 단색 모노 UI, CVSS 스타일의 10점 만점 스코어링, 순수 HTML/CSS/vanilla JS. 빌드 스텝도 프레임워크도 백엔드도 없습니다.

Vercel에 연결하면 `index.html`만 그대로 서빙하면 됩니다.

---

## 왜 만들었나

프로젝트 주제를 고를 때 사람들이 가장 자주 저지르는 실수는 **"해보고 싶다"와 "해도 되는 주제다"를 혼동하는 것**입니다. 아이디어는 머릿속에서 항상 멋있게 들립니다. 문제는 대부분의 아이디어가 이렇다는 점입니다.

- 이미 누가 했거나,
- 실제로 불편한 사람이 없거나,
- 기간 안에 도저히 끝낼 수 없거나,
- 완성해도 무엇을 두고 "끝났다"고 말해야 할지 본인도 모릅니다.

대학 캡스톤, 동아리 사이드 프로젝트, 대회·공모전, 학부 연구. 어디서든 주제 선정 단계에서 이 지점이 무너지면 이후 한두 학기가 통째로 낭비됩니다. **PROJECT:SCORE는 그 무너지는 지점을 한 페이지 안에서 먼저 마주하게 만들기 위해 존재합니다.**

## 설계 철학

### 1. 점수가 목적이 아니라 **대화의 시작점**입니다

10점 만점이 나와도 프로젝트 성공이 보장되지는 않습니다. 4점이 나와도 "배우려고" 모드라면 충분할 수 있습니다. 숫자는 판정이 아니라 **어떤 축이 약한지 한눈에 보여주는 지도**입니다. 축별 점수와 플래그·게이트 실패 여부가 숫자 자체보다 훨씬 중요합니다.

### 2. "단순 따라하기"가 항상 나쁜 것은 아닙니다

튜토리얼 복제는 포트폴리오용으로는 감점이지만, 학습용으로는 정당한 선택입니다. 그래서 **목적 모드(`배우려고` / `만들어보려고` / `보여주려고`)** 를 먼저 고르게 했습니다. 같은 주제라도 모드에 따라 다음이 달라집니다.

- 임계값이 달라집니다 (학습 모드는 5.5점이 Pass, 쇼케이스 모드는 8.0점이 Pass)
- 질문 가중치가 달라집니다 (학습 모드는 Novelty 축 비중이 절반)
- 일부 리젝트 플래그가 자동 해제됩니다 (학습 모드에서는 R1 "튜토리얼 복제"가 풀립니다)

### 3. IT 프로젝트는 "개발 대 연구" 이분법이 아닙니다

실제 IT 프로젝트는 훨씬 다양합니다. 앱 개발, 학술 연구, 버그 바운티, 오픈소스 기여, 기술 콘텐츠 제작. 각각 "누가 쓰는지", "결과물이 무엇인지", "어떻게 검증하는지"가 모두 다릅니다. 그래서 **프로젝트 성격을 복수 선택** 하도록 했습니다.

- `개발` — 앱·서비스·제품·도구 만들기
- `연구` — 실험·분석·조사·논문
- `보안` — 버그 바운티·취약점 분석·CTF·침투 테스트
- `기여` — 오픈소스 PR·커뮤니티·생태계 기여
- `콘텐츠` — 기술 글·강의·교재·튜토리얼

보안+개발, 개발+콘텐츠처럼 조합해서 고를 수도 있습니다. 성격에 따라 5개 핵심 질문(I2·I4·D1·D2·E1)의 문구가 분기됩니다.

### 4. 게이트와 질문은 **역할이 다릅니다**

게이트 5개는 점수로 채점되는 것이 아니라 **운영 측면의 하드 체크포인트**입니다. 시간 확보, 리뷰어, 기록 공간, 마감 일정, 중단 기준. 하나라도 "아직입니다"면 점수가 아무리 높아도 REVISE 판정이 뜹니다. 질문 14개는 주제 자체의 품질을, 게이트 5개는 프로젝트 운영 가능성을 다룹니다. 이 둘이 겹치지 않도록 설계했습니다.

### 5. 플래그는 **보너스가 아니라 경보**입니다

3개의 하드 리젝트 플래그(R1 튜토리얼 복제, R2 자원 부족, R3 법·윤리)와 3개의 감점 플래그(P1 과잉 범위, P2 검증 불가, P3 기록 없음)는 "혹시 이거 해당되나요?"를 묻는 자기 점검 체크리스트입니다. 해당되면 점수와 무관하게 RETHINK 또는 감점이 걸립니다.

### 6. 체스판 미학, **절제된 단색**

Material 3 다크 톤, 장식적 그라디언트, 컬러 액센트는 전부 배제했습니다. 검정(#0a0a0b), 흰색(#fafafa), 그 사이의 회색 네 단계만 사용합니다. 보더는 얇고 일관된 1px, 그리드는 엄격한 8px 단위, 타이포는 IBM Plex Mono(라틴)와 Pretendard·나눔명조(한글)만 씁니다. 이 디자인 언어는 아이디어에 감정을 덧칠하지 않고 **숫자와 문장만 남기는 것**을 목표로 합니다.

## 점수 체계 (10점 만점, CVSS 스타일)

### 축 (5개)

| 코드 | 이름 | 한국어 | 최대 가중치 |
|---|---|---|---:|
| I | Impact | 왜 만드는가 | 25 |
| N | Novelty | 무엇이 새로운가 | 20 |
| F | Feasibility | 할 수 있는가 | 20 |
| D | Deliverables | 무엇을 내놓는가 | 20 |
| E | Evidence | 정말로 되는가 | 15 |

### 점수 계산

```
questionScore = (답한 가중치 합 / 전체 가중치) × 8
gateScore     = (통과 게이트 수 / 5) × 2
scaled        = questionScore + gateScore                  // 0–10
total         = max(0, scaled − 감점 플래그 합계)            // 0–10
```

질문이 80%, 게이트가 20%를 차지합니다. 감점 플래그가 켜지면 최종 점수에서 차감됩니다.

### 판정 임계값 (모드별)

| 모드 | PASS | ALMOST | LEARN FIRST | 그 외 |
|---|---:|---:|---:|---|
| 🎒 배우려고 | 5.5 | 4.0 | 2.5 | RETHINK |
| 🛠 만들어보려고 | 7.2 | 5.8 | 4.3 | RETHINK |
| 🏆 보여주려고 | 8.0 | 6.5 | 5.0 | RETHINK |

단, 다음은 점수와 무관하게 판정됩니다.

- **하드 리젝트 플래그가 켜진 경우** → 즉시 `RETHINK`
- **실패한 게이트가 있는 경우** → 즉시 `REVISE`

## 사용법

1. **목적 모드 고르기**: 배우려고 / 만들어보려고 / 보여주려고
2. **프로젝트 성격 고르기** (복수 선택 가능, 아무것도 고르지 않아도 됩니다)
3. **5개 게이트 토글**: 예 / 아직입니다
4. **14개 질문**: 각 질문에 0점(없음·모르겠음) / 1점(일부) / 2점(확실)
5. **플래그 체크**: 해당하는 리젝트·감점 항목을 체크
6. **결과 저장**: Markdown(사람이 읽기 쉬움) 또는 JSON(기계용)

새로 고칠 때마다 응답이 초기화되도록 페이지 로드 시 `localStorage`를 비웁니다. 어차피 이 데이터는 브라우저에만 머무르고 서버로 보내지 않습니다.

## 로컬 실행

정적 페이지라서 서버만 있으면 됩니다.

```bash
python3 -m http.server 8000
# http://localhost:8000 접속
```

`index.html`을 브라우저로 바로 열어도 동작합니다. ES 모듈 import 때문에 파일 프로토콜이 막히면 위 명령을 사용하세요.

## 배포 (Vercel)

1. 이 레포를 Vercel에 import
2. Framework Preset: `Other`
3. Build Command: 비워둠
4. Output Directory: 비워둠 (루트의 `index.html` 서빙)
5. Deploy

`main` 브랜치에 푸시할 때마다 Vercel이 자동으로 재빌드합니다.

## 파일 구조

```
.
├── index.html       # 전체 마크업 (섹션 1~4 + 스코어 카드 + 액션)
├── styles.css       # v1.2 체스판 모노 테마, 8px 그리드, 3단 반응형
├── app.js           # 상태·스코어링·렌더링·이벤트·내보내기
├── questions.js     # 데이터: 모드·타입·축·질문·게이트·플래그·판정
└── README.md        # 이 문서
```

`app.js`와 `questions.js`는 ES 모듈로 분리해서 데이터 튜닝과 로직 리팩토링이 섞이지 않도록 했습니다. 질문 문구·가중치·임계값을 바꾸고 싶다면 `questions.js`만 건드리면 됩니다.
